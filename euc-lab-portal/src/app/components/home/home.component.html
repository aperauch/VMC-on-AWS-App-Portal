Getting Started
Use this tool to allow and manage public Internet access to your workloads by followings these steps:

  1. Request a public IP for your workloads.
  2. Create a public DNS record for your new IP using the psolab.net, euclab.net, or even your personal domain.
  3. Create a NAT rule to map your public IP to a load balancer virtual IP (VIP) or the real IP of your target server.
  4. Optionally, review the existing Compute Gateway (CGW) firewall rules to ensure application specific traffic is allowed.
  5. Download a TLS web certificate for you application server if needed.
